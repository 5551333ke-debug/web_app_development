# 活動報名系統 PRD 文件

## 1. 專案概述

### 背景與動機
在舉辦各種活動（如企業講座、校園社團活動或個人主辦的交流會）時，報名資料的收集與管理往往耗費主辦單位大量心力。傳統的表單工具雖然方便，但缺乏專門針對活動流程優化的功能，如自動提醒、電子票券及現場快速簽到等。本專案旨在提供一個整合性的解決方案，簡化報名流程並提升管理效率。

### 目標用戶
- **活動主辦方**：企業、學校、社團或個人主辦人，需要建立活動並自訂報名規則。
- **一般參與者**：想報名活動的使用者，追求簡單、快速且流暢的報名體驗。
- **活動管理人員**：負責名單管理、報名狀況追蹤以及活動現場的簽到執行。

### 核心價值主張
- **快速建立**：主辦方可在數分鐘內建立一個具備專業外觀的報名頁面。
- **自訂彈性**：支援自訂報名欄位，滿足不同活動的資訊收集需求。
- **高效管理**：整合通知系統與電子票券，自動化重複性的行政工作，提升專業度。

## 2. 功能需求

| 功能名稱 | 使用者故事 (User Story) |
| :--- | :--- |
| **活動建立與編輯** | 作為**主辦方**，我希望能夠設定活動名稱、時間、地點、名額與報名期限，以便讓參與者了解活動詳情並進行報名。 |
| **線上報名表單** | 作為**主辦方**，我希望能夠自訂報名表單欄位（如姓名、Email、電話等），以便收集特定活動所需的參與者資訊。 |
| **報名名單管理** | 作為**活動管理人員**，我希望能夠查看、搜尋與匯出報名資料，以便掌握報名進度與參與者資訊。 |
| **通知系統** | 作為**一般參與者**，我希望在報名成功後收到確認信，並在活動前收到提醒通知，以便確認我的名額已保留且不會錯過活動。 |
| **電子票券與簽到** | 作為**活動管理人員**，我希望系統能產生 QR Code 並支援現場掃碼報到，以便提高現場報到效率並精準記錄出席狀況。 |

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask。
- **模板引擎**：使用 Jinja2。
- **資料庫**：使用 SQLite，便於開發與輕量部署。

### 效能與安全考量
- **響應式設計**：報名頁面必須在行動裝置上運作流暢。
- **資料驗證**：確保 Email 格式正確且必填欄位不留白。
- **資料匯出安全性**：限制僅管理人員可匯出參與者清單。

## 4. MVP 範圍

### Must Have (必須完成)
- 活動建立、編輯與顯示功能。
- 基礎報名表單（姓名、Email、電話）。
- 後台查看報名清單功能。
- 報名成功自動寄送確認信（模擬或實際整合）。

### Should Have (應該包含)
- 自訂報名表單欄位。
- 名單匯出為 CSV 功能。
- QR Code 電子票券產生。

### Nice to Have (進階加分)
- 活動現場手機掃碼簽到介面。
- 活動倒數提醒通知。
- 活動報名統計圖表。
