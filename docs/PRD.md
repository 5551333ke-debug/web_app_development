# 產品需求文件 (PRD) - 活動報名系統

## 1. 專案概述

### 背景與動機
在現今數位化的時代，活動主辦單位（如企業、學校、社團）需要一個高效且易於使用的平台來管理活動報名流程。傳統的紙本或簡單表單往往難以處理自訂欄位、自動通知及現場簽到等複雜需求。本專案旨在建立一個一站式的活動報名與管理系統，簡化從活動建立到現場執行的完整流程。

### 目標用戶
1.  **活動主辦方**：負責建立活動、設定報名規則與自訂表單的組織或個人。
2.  **一般參與者**：希望快速瀏覽活動資訊並完成報名的使用者。
3.  **活動管理人員**：在活動現場或後台負責審核名單、匯出資料及執行現場簽到的人員。

### 核心價值主張
- **快速部署**：主辦方可在幾分鐘內建立一個專業的報名頁面。
- **靈活自訂**：支援自訂報名欄位，滿足不同類型的活動需求。
- **自動化維運**：自動化郵件通知與 QR Code 簽到系統，節省人力成本。

## 2. 功能需求

### 主要功能與使用者故事

1.  **活動建立與編輯**
    - **使用者故事**：作為 **活動主辦方**，我希望能夠 **設定活動的名稱、時間、地點、名額與報名期限**，以便 **讓潛在參與者了解活動詳情並在時限內報名**。
    - **細節**：包含活動主圖上傳、富文本描述編輯、報名狀態切換（開啟/關閉）。

2.  **線上報名表單**
    - **使用者故事**：作為 **一般參與者**，我希望能夠 **填寫包含姓名、Email、電話及主辦方指定的自訂欄位**，以便 **完成報名程序**。
    - **細節**：支援必填/選填設定，前端欄位格式校驗（如 Email 格式）。

3.  **報名名單管理**
    - **使用者故事**：作為 **活動管理人員**，我希望能夠 **查看、搜尋並匯出報名名單**，以便 **掌握報名狀況並進行後續的行政處理**。
    - **細節**：支援關鍵字搜尋（姓名/Email），提供 CSV/Excel 格式匯出功能。

4.  **通知系統**
    - **使用者故事**：作為 **一般參與者**，我希望在 **報名成功後收到確認信**，以及在 **活動前收到提醒通知**，以便 **確保我不會錯過活動**。
    - **細節**：自動發送報名完成通知，後台可觸發活動前提醒信。

5.  **電子票券與現場簽到**
    - **使用者故事**：作為 **活動管理人員**，我希望能夠 **掃描參與者的 QR Code 進行報到**，以便 **快速完成現場簽到並減少排隊時間**。
    - **細節**：報名成功信中附帶唯一 QR Code，手機瀏覽器可開啟掃碼頁面進行即時核銷。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python 3.x)
- **前端模板**：Jinja2 + Vanilla CSS
- **資料庫**：SQLite (開發與初期部署)
- **部署環境**：支援常見的雲端平台（如 Render, Heroku 或私人伺服器）

### 效能與安全考量
- **響應式設計**：確保報名頁面在手機與電腦上皆有良好體驗。
- **資料安全**：保護參與者的隱私資訊，防止未經授權的資料擷取。
- **效能**：支援至少 100 人同時在線報名的併發處理能力。
